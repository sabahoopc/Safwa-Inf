# دليل مؤثري صفوى

منصة مجتمعية لإبراز المؤثرين في مدينة صفوى — مرشَّحون من الجمهور ومُقيَّمون بمعايير أخلاقية واضحة.

---

## هيكل المشروع

```
safwa-directory/
├── index.html          # الصفحة الرئيسية — عرض الدليل
├── nominate.html       # صفحة ترشيح مؤثر
├── remove.html         # صفحة طلب إزالة مؤثر
├── admin/
│   └── index.html      # لوحة إدارة الدليل (محمية بكلمة مرور)
├── css/
│   ├── main.css        # التصميم العام (layout, header, hero, buttons)
│   ├── cards.css       # بطاقات المؤثرين
│   ├── forms.css       # نماذج الترشيح والإزالة
│   └── admin.css       # لوحة الإدارة
└── js/
    ├── data.js         # المخزن المركزي للبيانات + localStorage
    ├── icons.js        # أيقونات SVG + دوال مساعدة
    ├── cards.js        # رسم بطاقات المؤثرين والإحصائيات
    ├── filters.js      # فلترة المؤثرين حسب المجال
    ├── modal.js        # نافذة تأكيد الإزالة
    ├── forms.js        # منطق نماذج الترشيح والإزالة
    └── admin.js        # منطق لوحة الإدارة الكاملة
```

---

## الميزات

### الدليل العام (`index.html`)
- عرض بطاقات المؤثرين مع فلترة حسب المجال
- رفع صورة شخصية لكل مؤثر
- روابط مباشرة للحسابات (انستجرام، سناب، إكس، يوتيوب)
- معايير الإدراج الأخلاقية
- زر طلب الإزالة على كل بطاقة

### ترشيح مؤثر (`nominate.html`)
- نموذج كامل مع التحقق من المدخلات
- رفع صورة للمُرشَّح
- الهوية مخفية (لا يُطلب اسم المُرشِّح)

### طلب إزالة (`remove.html`)
- تحديد المؤثر والسبب والشرح التفصيلي
- رفع أدلة (روابط أو وصف)
- الهوية مخفية
- دعم Pre-select عبر URL param: `?target=اسم_المؤثر`

### لوحة الإدارة (`admin/index.html`)
- تسجيل دخول بكلمة مرور
- إدارة المؤثرين (إضافة / تعديل / حذف)
- مراجعة طلبات الترشيح (قبول → يُضاف تلقائياً للدليل | رفض)
- مراجعة طلبات الإزالة مع الأدلة (تنفيذ | رفض)
- عداد البلاغات لكل مؤثر

## التشغيل المحلي

لا يحتاج المشروع server — افتح `index.html` مباشرةً في المتصفح.

```bash
# أو استخدم live server بسيط
npx serve .
```

---

## النشر على GitHub Pages

1. ارفع المجلد على GitHub repo
2. اذهب إلى **Settings → Pages**
3. اختر **Branch: main** و **Folder: / (root)**
4. الموقع سيكون متاحاً على: `https://username.github.io/safwa-directory`

## البيانات

حالياً البيانات تُخزَّن في `localStorage` في المتصفح (مناسب للنماذج والتجربة).

### للإنتاج الفعلي:
استبدل دوال `data.js` بـ API calls لأحد هذه الخيارات:
- **Firebase Firestore** — مجاني للمشاريع الصغيرة، سهل الإعداد
- **Supabase** — بديل مفتوح المصدر لـ Firebase
- **Google Sheets API** — بسيط جداً لفريق صغير
